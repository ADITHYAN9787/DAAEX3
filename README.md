# Implementation of Kruskal's and Prim's Algorithms for Minimum Spanning Tree

## 📌 Overview

This project implements two popular Minimum Spanning Tree (MST) algorithms:

- **Kruskal's Algorithm**
- **Prim's Algorithm**

Both algorithms are applied to the same weighted, undirected graph containing **7 vertices (0–6)** and **9 edges**. The project compares the MST generated by each algorithm, verifies that both produce the same minimum total weight, and discusses their time complexities.

---

## 📖 Problem Statement

Given a weighted undirected graph with **7 vertices (0–6)** and **9 edges**, implement **Kruskal's Algorithm** and **Prim's Algorithm** to find the **Minimum Spanning Tree (MST)**.

Verify that:

- Both algorithms produce the same minimum spanning tree weight.
- The resulting MST contains exactly **V − 1** edges.
- The time complexity of both algorithms is analyzed and compared.

---

## 🎯 Objectives

- Implement Kruskal's Algorithm.
- Implement Prim's Algorithm.
- Construct the Minimum Spanning Tree (MST).
- Compare the total weight obtained by both algorithms.
- Analyze their time complexity.

## 📚 Algorithms Used

### Kruskal's Algorithm

Kruskal's Algorithm constructs the MST by repeatedly selecting the smallest edge that does not form a cycle.

**Technique Used**
- Greedy Algorithm
- Disjoint Set (Union-Find)

**Time Complexity**

| Operation | Complexity |
|-----------|------------|
| Sorting Edges | O(E log E) |
| Union-Find Operations | O(E α(V)) |
| Overall | **O(E log E)** |

---

### Prim's Algorithm

Prim's Algorithm starts from any vertex and repeatedly adds the minimum weight edge connecting the growing MST to an unvisited vertex.

**Technique Used**
- Greedy Algorithm
- Priority Queue (Min Heap)

**Time Complexity**

| Data Structure | Complexity |
|---------------|------------|
| Min Heap | O(E log V) |
